# 智能车速度发布节点 - 添加键盘停止功能教程

## 一、教程目的

本教程将指导您如何修改现有的智能车速度发布节点，添加键盘'q'键停止功能。通过本教程，您将学习：

1. ROS2节点中如何集成键盘输入监听
2. 多线程编程在ROS2中的应用
3. 安全停止机制的设计与实现
4. 代码重构与模块化设计

## 二、修改前的代码分析

### 当前文件：`src/smart_car_core_demo/src/speed_publisher.cpp`

```cpp
#include <chrono>

#include "geometry_msgs/msg/twist.hpp"
#include "rclcpp/rclcpp.hpp"
using namespace std::chrono_literals;

/**
 * @brief 智能车车速发布节点
 *
 * 功能：定时发布车速指令到/cmd_vel话题
 * 对应智能车场景：控制电机运动
 */
class SpeedPublisher : public rclcpp::Node {
   public:
    SpeedPublisher() : Node("speed_publisher") {
        publisher_ = this->create_publisher<geometry_msgs::msg::Twist>("cmd_vel", 10);
        timer_ = this->create_wall_timer(
            100ms, 
            std::bind(&SpeedPublisher::timer_callback, this)
        );
    }

   private:  // ✅ 私有成员，保护封装性
    void timer_callback() {
        auto message = geometry_msgs::msg::Twist();
        message.linear.x = 0.5;  // 设置线速度为0.5 m/s
        message.linear.y = 0.0; 
        message.linear.z = 0.0;
        message.angular.x = 0.0;
        message.angular.y = 0.0;
        message.angular.z = 0.2; // 设置角速度为0.2 rad/s
        publisher_->publish(message);

        // 打印日志
        RCLCPP_INFO(this->get_logger(),
                    "发布车速度指令：线速度=%.2f m/s, 角速度=%.2f rad/s",
                    message.linear.x, message.angular.z);
    }
    rclcpp::Publisher<geometry_msgs::msg::Twist>::SharedPtr publisher_;
    rclcpp::TimerBase::SharedPtr timer_;
};

int main(int argc, char *argv[]){
    rclcpp::init(argc,argv);
    rclcpp::spin(std::make_shared<SpeedPublisher>());
    rclcpp::shutdown();
    return 0;
}
```

### 当前功能限制：
1. 只能通过Ctrl+C强制停止节点
2. 停止时可能还在发布速度指令，存在安全隐患
3. 缺乏用户友好的交互方式

## 三、修改方案设计

### 设计原则：
1. **安全性优先**：停止前发布零速度指令，防止智能车失控
2. **非阻塞设计**：键盘监听不干扰ROS2事件循环
3. **代码清晰**：保持原有结构，新增功能模块化
4. **易于扩展**：为未来添加更多键盘控制功能预留接口

### 技术方案：
- 使用`std::thread`创建独立键盘监听线程
- 使用`std::atomic<bool>`实现线程安全的状态标志
- 使用Linux终端原始模式实现非阻塞键盘输入

```cpp
// 实现：非阻塞获取键盘输入（Linux终端原始模式）
char SpeedPublisher::getch() {
    char buf = 0;
    struct termios old = {0};
    
    // 保存当前终端设置
    if (tcgetattr(0, &old) < 0)
        perror("tcsetattr()");
    
    // 设置终端为原始模式（非阻塞）
    old.c_lflag &= ~ICANON;
    old.c_lflag &= ~ECHO;
    old.c_cc[VMIN] = 1;
    old.c_cc[VTIME] = 0;
    
    if (tcsetattr(0, TCSANOW, &old) < 0)
        perror("tcsetattr ICANON");
    
    // 读取一个字符
    if (read(0, &buf, 1) < 0)
        perror("read()");
    
    // 恢复终端设置
    old.c_lflag |= ICANON;
    old.c_lflag |= ECHO;
    
    if (tcsetattr(0, TCSADRAIN, &old) < 0)
        perror("tcsetattr ~ICANON");
    
    return buf;
}

// 实现：发布零速度指令
void SpeedPublisher::publishZeroVelocity() {
    auto zero_msg = geometry_msgs::msg::Twist();
    zero_msg.linear.x = 0.0;
    zero_msg.linear.y = 0.0;
    zero_msg.linear.z = 0.0;
    zero_msg.angular.x = 0.0;
    zero_msg.angular.y = 0.0;
    zero_msg.angular.z = 0.0;
    
    publisher_->publish(zero_msg);
    RCLCPP_INFO(this->get_logger(), "已发布零速度指令，安全停止");
}

// 实现：键盘监听线程
void SpeedPublisher::keyboardListener() {
    RCLCPP_DEBUG(this->get_logger(), "键盘监听线程已启动");
    
    while (rclcpp::ok() && !should_stop_) {
        char c = getch();  // 获取键盘输入
        
        if (c == 'q' || c == 'Q') {
            RCLCPP_INFO(this->get_logger(), "收到停止指令，正在安全停止...");
            
            // 1. 发布零速度指令
            publishZeroVelocity();
            
            // 2. 设置停止标志
            should_stop_ = true;
            
            // 3. 通知ROS2系统停止
            rclcpp::shutdown();
            
            RCLCPP_INFO(this->get_logger(), "节点停止指令已发送");
            break;
        }
        
        // 短暂休眠，避免CPU占用过高
        std::this_thread::sleep_for(10ms);
    }
    
    RCLCPP_DEBUG(this->get_logger(), "键盘监听线程已退出");
}
```

## 七、常见问题与解决方案

### 问题1：编译错误 "termios.h: No such file or directory"
**原因**：Linux特有头文件，在非Linux系统上不可用
**解决方案**：本代码仅适用于Linux系统。如果您在其他系统上，需要修改`getch()`函数实现。

### 问题2：按下`q`键无反应
**原因**：
1. 终端焦点不在运行节点的窗口
2. 键盘输入被其他程序拦截
3. 终端模式设置失败
**解决方案**：
1. 确保点击节点运行窗口使其获得焦点
2. 关闭可能拦截键盘输入的程序
3. 检查终端权限，尝试在普通终端而非IDE内置终端中运行

### 问题3：节点停止后进程未完全退出
**原因**：线程未正确join
**解决方案**：确保在析构函数中调用`keyboard_thread_.join()`

### 问题4：发布零速度后电机仍在转动
**原因**：电机控制器有惯性或延迟
**解决方案**：
1. 连续发布多次零速度指令
2. 增加停止延迟
3. 修改`publishZeroVelocity()`函数：

```cpp
void SpeedPublisher::publishZeroVelocity() {

    zero_msg.linear.x = 0.0;
    zero_msg.linear.y = 0.0;
    zero_msg.linear.z = 0.0;

    zero_msg.angular.y = 0.0;
    zero_msg.angular.z = 0.0;
    
    // 连续发布3次，确保电机控制器收到
    for (int i = 0; i < 3; i++) {
        publisher_->publish(zero_msg);
        std::this_thread::sleep_for(50ms);
    }
    
    RCLCPP_INFO(this->get_logger(), "已发布零速度指令，安全停止");
}
```

```cpp
} else if (c == 'a' || c == 'A') {
    // 左转
    current_angular_ += 0.1;
} else if (c == 'd' || c == 'D') {
    // 右转
    current_angular_ -= 0.1;
} else if (c == ' ') {
    // 空格键紧急停止
    emergencyStop();
}
```
