# 手动启动智能车速度控制节点教程

## 概述

本教程介绍如何手动启动智能车速度控制节点。包含两种方法：
1. **手动启动单个节点** - 分别启动发布者和订阅者节点
2. **使用launch文件启动** - 一次性启动所有相关节点

## 环境准备

### 1. 设置ROS2环境
```bash
# 如果使用bash
source /opt/ros/humble/setup.bash

# 如果使用zsh
source /opt/ros/humble/setup.zsh

# 设置工作空间环境
cd /home/ubuntu/ros2_wsorincp
source install/setup.bash
```

### 2. 检查已编译的节点
```bash
# 查看smart_car_core_demo包中的可执行文件
ros2 pkg executables smart_car_core_demo

# 预期输出：
# smart_car_core_demo speed_publisher
# smart_car_core_demo speed_subscriber_simple
```

## 方法一：手动启动单个节点

### 步骤1：启动速度发布节点
```bash
# 在一个终端中启动速度发布节点
ros2 run smart_car_core_demo speed_publisher
```

**预期输出：**
```
[INFO] [speed_publisher]: 发布车速度指令：线速度=0.50 m/s, 角速度=0.20 rad/s
[INFO] [speed_publisher]: 发布车速度指令：线速度=0.50 m/s, 角速度=0.20 rad/s
[INFO] [speed_publisher]: 发布车速度指令：线速度=0.50 m/s, 角速度=0.20 rad/s
...
```
*每100ms发布一次速度指令*

### 步骤2：启动速度订阅节点（简化版）
```bash
# 在另一个终端中启动速度订阅节点
ros2 run smart_car_core_demo speed_subscriber_simple
```

**预期输出：**
```
[INFO] [speed_subscriber_simple]: 简化版速度订阅节点已启动，等待cmd_vel指令...
[INFO] [speed_subscriber_simple]: 收到速度指令: 线速度(0.50, 0.00) m/s, 角速度0.20 rad/s
[INFO] [speed_subscriber_simple]: 计算电机转速: 1:1.64, 2:1.64, 3:-1.64, 4:-1.64 rps
...
```

### 步骤3：验证节点通信
```bash
# 在新的终端中查看话题列表
ros2 topic list

# 预期输出包含：
# /cmd_vel
# /motor_speeds
# /parameter_events
# /rosout

# 查看/cmd_vel话题的消息
ros2 topic echo /cmd_vel

# 查看/motor_speeds话题的消息
ros2 topic echo /motor_speeds
```

## 方法二：使用launch文件启动

### 步骤1：启动launch文件
```bash
# 在单个命令中启动所有节点
ros2 launch smart_car_core_demo topic_demo.launch.py
```

**预期输出：**

## 节点功能说明

### 1. speed_publisher 节点
- **功能**：定时发布车速指令到 `/cmd_vel` 话题
- **发布频率**：10Hz（每100ms）
- **速度参数**：
  - 线速度：0.5 m/s（前进）
  - 角速度：0.2 rad/s（小转弯）
- **消息类型**：`geometry_msgs/Twist`

### 2. speed_subscriber_simple 节点
- **功能**：订阅 `/cmd_vel` 话题，计算电机转速
- **运动学模型**：麦克纳姆轮底盘
- **输出**：
  - 发布电机转速到 `/motor_speeds` 话题
  - 打印计算后的电机转速到控制台
- **消息类型**：`std_msgs/Float32MultiArray`

## 麦克纳姆轮运动学计算

### 运动学公式
```
motor1 = linear_x - linear_y - angular_z * factor
motor2 = linear_x + linear_y - angular_z * factor  
motor3 = linear_x + linear_y + angular_z * factor
motor4 = linear_x - linear_y + angular_z * factor
```

其中：
- `linear_x`: X方向线速度 (m/s)
- `linear_y`: Y方向线速度 (m/s)  
- `angular_z`: Z方向角速度 (rad/s)
- `factor`: (轴距 + 轮距) / 2

### 参数说明
- **轴距**：0.216 m（前后轮距离）
- **轮距**：0.195 m（左右轮距离）
- **轮子直径**：0.097 m

## 故障排除

### 常见问题1：节点无法启动
```bash
# 检查环境设置
echo $ROS_DISTRO  # 应该显示 "humble"

# 检查包是否已编译
ros2 pkg list | grep smart_car_core_demo

# 重新编译包
cd /home/ubuntu/ros2_wsorincp
colcon build --packages-select smart_car_core_demo
source install/setup.bash
```

### 常见问题2：话题无法通信
```bash
# 检查话题是否正常发布
ros2 topic info /cmd_vel

# 检查节点是否正常运行
ros2 node list

# 检查节点详细信息
ros2 node info /speed_publisher
ros2 node info /speed_subscriber_simple
```

### 常见问题3：消息类型不匹配
```bash
# 检查消息类型
ros2 topic type /cmd_vel  # 应该显示 "geometry_msgs/msg/Twist"
ros2 topic type /motor_speeds  # 应该显示 "std_msgs/msg/Float32MultiArray"

# 查看消息结构
ros2 interface show geometry_msgs/msg/Twist
ros2 interface show std_msgs/msg/Float32MultiArray
```

## 扩展练习

### 练习1：修改速度参数
```bash
# 手动发布不同的速度指令
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.3, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.1}}" -1
```

### 练习2：实时监控话题
```bash
# 同时监控多个话题
ros2 topic hz /cmd_vel /motor_speeds

# 查看话题带宽使用
ros2 topic bw /cmd_vel
```

### 练习3：记录和回放数据
```bash
# 记录话题数据
ros2 bag record /cmd_vel /motor_speeds

# 回放记录的数据
ros2 bag play <bag_file_name>
```
