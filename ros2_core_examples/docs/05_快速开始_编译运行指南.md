# 快速开始指南 - 智能车ROS2核心代码编译运行

## 概述
本指南提供完整的编译和运行步骤，帮助您快速在智能车上部署ROS2四大核心代码。

## 1. 环境准备

### 1.1 系统要求
- Ubuntu 22.04 或更高版本
- ROS2 Humble 或 Jazzy 版本
- C++17 编译器
- 智能车硬件（可选，可在PC上模拟运行）

### 1.2 检查ROS2环境
```bash
# 检查ROS2版本
ros2 --version

# 检查环境变量
printenv | grep ROS

# 测试ROS2基础功能
ros2 topic list
```

## 2. 创建和配置项目

### 2.1 创建工作空间
```bash
# 进入您的工作空间目录
cd ~/ros2_ws/src

# 创建智能车演示包
ros2 pkg create --build-type ament_cmake smart_car_core_demo \
  --dependencies rclcpp geometry_msgs std_msgs std_srvs rclcpp_action
```

### 2.2 复制代码文件
将本目录中的代码文件复制到相应位置：
```bash
# 复制源代码文件
cp -r src/Myself_WS/ros2_core_examples/src/smart_car_core_demo/src/* ~/ros2_ws/src/smart_car_core_demo/src/

# 复制接口定义文件
mkdir -p ~/ros2_ws/src/smart_car_core_demo/srv
mkdir -p ~/ros2_ws/src/smart_car_core_demo/action
mkdir -p ~/ros2_ws/src/smart_car_core_demo/launch

# 复制CMakeLists.txt和package.xml
cp src/Myself_WS/ros2_core_examples/CMakeLists.txt ~/ros2_ws/src/smart_car_core_demo/
cp src/Myself_WS/ros2_core_examples/package.xml ~/ros2_ws/src/smart_car_core_demo/
```

## 3. 编译项目

### 3.1 安装依赖
```bash
# 更新包列表
sudo apt update

# 安装nlohmann-json3-dev（用于JSON处理）
sudo apt install nlohmann-json3-dev

# 安装ROS2开发工具
sudo apt install ros-${ROS_DISTRO}-rosidl-default-generators
```

### 3.2 编译项目
```bash
# 进入工作空间根目录
cd ~/ros2_ws

# 编译智能车演示包
colcon build --packages-select smart_car_core_demo

# 如果编译成功，设置环境变量
source install/setup.bash
```

### 3.3 常见编译问题解决

#### 问题1：找不到nlohmann/json.hpp
```bash
# 安装nlohmann-json库
sudo apt install nlohmann-json3-dev

# 在CMakeLists.txt中添加
find_package(nlohmann_json REQUIRED)
```

#### 问题2：ROS2接口生成失败
```bash
# 确保已安装rosidl生成器
sudo apt install ros-${ROS_DISTRO}-rosidl-default-generators

# 在package.xml中添加
<build_depend>rosidl_default_generators</build_depend>
<exec_depend>rosidl_default_runtime</exec_depend>
<member_of_group>rosidl_interface_packages</member_of_group>
```

#### 问题3：C++标准不匹配
```cmake
# 在CMakeLists.txt中设置C++17
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
```

## 5. 在智能车上实际部署

### 5.1 硬件连接检查
```bash
# 检查USB设备连接
lsusb

# 检查串口设备
ls /dev/ttyUSB*
ls /dev/ttyACM*

# 检查网络连接
ip addr show
```

### 5.2 配置智能车参数
创建配置文件 `config/smart_car_params.yaml`：
```yaml
integrated_control:
  ros__parameters:
    # 控制参数
    control_frequency: 10.0      # 控制频率(Hz)
    max_speed: 1.0               # 最大速度(m/s)
    min_speed: 0.1               # 最小速度(m/s)
    max_angular_speed: 1.0       # 最大角速度(rad/s)
    
    # 安全参数
    emergency_stop_timeout: 5.0  # 紧急停止超时(秒)
    battery_warning_level: 11.5  # 电池警告电压(V)
    battery_critical_level: 11.0 # 电池临界电压(V)
    
    # 动作参数
    countdown_min_seconds: 1     # 最小倒计时秒数
    countdown_max_seconds: 60    # 最大倒计时秒数
```

### 5.3 启动智能车系统
```bash
# 使用launch文件启动所有节点
ros2 launch smart_car_core_demo smart_car.launch.py

# 或手动启动整合节点
ros2 run smart_car_core_demo integrated_control_node --ros-args --params-file config/smart_car_params.yaml
```

## 7. 性能优化建议

### 7.1 提高实时性
```cpp
// 使用实时优先级（需要sudo权限）
#include <pthread.h>
pthread_t this_thread = pthread_self();
struct sched_param params;
params.sched_priority = sched_get_priority_max(SCHED_FIFO);
pthread_setschedparam(this_thread, SCHED_FIFO, &params);
```

### 7.2 优化QoS设置
```cpp
// 对于控制话题，使用可靠的QoS
auto qos = rclcpp::QoS(rclcpp::KeepLast(10)).reliable();
publisher_ = this->create_publisher<geometry_msgs::msg::Twist>("/cmd_vel", qos);

// 对于传感器数据，使用尽力而为的QoS
auto sensor_qos = rclcpp::SensorDataQoS();
subscription_ = this->create_subscription<std_msgs::msg::Float32>(
    "/sensor_data", sensor_qos, callback);
```

### 7.3 内存管理
```cpp
// 使用智能指针避免内存泄漏
std::shared_ptr<Message> msg = std::make_shared<Message>();

// 预分配内存池
std::vector<std::shared_ptr<Message>> message_pool(100);
```
